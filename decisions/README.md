# Why Document LLM Decisions?

LLM selection is not a one-time event. Models change, pricing shifts, and new options emerge quarterly.

Without documentation, teams face:
- **No institutional memory** — nobody remembers why Model X was chosen 6 months ago
- **Repeated evaluation work** — new team members re-run the same comparisons
- **No audit trail** — compliance and leadership can't see the rationale
- **Risky upgrades** — no baseline to compare against when switching models

Every model selection should produce a decision record using `template.md`.
Store completed records in `examples/` with the naming convention `YYYY-MM-DD_description.md`.

Review decisions quarterly or when a model provider announces significant changes.
